# LLaVA — Visual Instruction Tuning（LLM＋線形projectorの原点）

画像を「見て指示に従って答える」汎用アシスタントを、専用の巨大設計なしに作るにはどうするか。LLaVA の答えは拍子抜けするほど単純で、**凍結した CLIP 視覚エンコーダ**の特徴を**たった 1 枚の線形射影行列 W** で言語モデル（Vicuna）の単語埋め込み空間へ写し、あとは言語モデル本体に視覚的推論を任せる、というものだ。その単純さを成立させた裏側の主役は、**text-only の GPT-4 に画像を「記号」で説明させて指示データを合成する**というデータ中心のアイデアである。一言で言えば、LLaVA は「軽いコネクタ＋強い LLM＋合成指示データ」で視覚対話を実現し、以後の多くの VLM の雛形を確立した論文だ。

## 全体像（まず一枚で）

LLaVA のネットワークは 3 つの部品しかない。画像を意味ベクトルに変える**視覚エンコーダ**、そのベクトルを言語側に橋渡しする**コネクタ（線形射影 W）**、そして橋を渡ってきた視覚トークンと言語指示をまとめて読み、応答を自己回帰生成する**言語モデル**である。データの流れは下から上へ一本道だ。

## コネクタ設計：単一の線形射影

LLaVA のコネクタは、行列 1 つの線形写像にすぎない。式で書けば

> H_v = W · Z_v,  ただし Z_v = g(X_v)

ここで g(·) は凍結された CLIP、W は学習対象の射影行列で、出力 H_v は LLM の単語埋め込みと**同じ次元**を持つ。クロスアテンションも、Q-Former のような追加 Transformer も挟まない。

なぜ線形 1 枚で足りるのか。直観はこうだ。CLIP は画像とテキストを対照学習で**すでに近い意味空間**へ揃えてある。だから視覚特徴は「言語に近い座標系」へ少し回転・スケーリングしてやれば十分で、その変換は線形写像で表せる範囲に収まる。そして本当に難しい「視覚情報をことばで推論する」処理は、後段の Vicuna の何十層もの Transformer が担う。コネクタは橋であって脳ではない、という役割分担だ。

論文自身も、この単純さを「軽量でデータ中心の実験を素早く回せる」利点として明言しており、Flamingo のゲート付きクロスアテンションや BLIP-2 の Q-Former のような凝った接続は**今後の課題**として残している。つまり線形射影は「最弱だが十分」を狙った設計上の意思決定であって、後継研究（LLaVA-1.5 の MLP コネクタなど）の出発点でもある。

### Stage 1：特徴整列（projector のみ学習）

第 1 段階は「特徴の整列（feature alignment）」と呼ばれる。CC3M をフィルタした**約 595K の画像-キャプション対**を、「この画像を簡潔に説明して」という単純な指示形式の単ターン会話に変換して使う。視覚エンコーダと LLM の重みは凍結し、学習するのは射影行列 W だけ。狙いは、H_v が**凍結済み LLM の単語埋め込みと噛み合う**ように W を整えること。論文はこの段階を「凍結 LLM 用の互換な視覚トークナイザを訓練する」と表現している。

### Stage 2：visual instruction tuning（projector＋LLM）

第 2 段階で、視覚エンコーダは凍結のまま、**W と LLM の両方**を更新する（学習パラメータ θ = {W, φ}）。ここで使うのが次節で述べる **GPT-4 合成の約 158K 指示データ（LLaVA-Instruct-158K）**だ。この end-to-end 微調整によって、モデルは単なる画像説明にとどまらず、ユーザの指示の意図をくみ取って多ターンで応答できる「アシスタント」へと変わる。論文では応用先として**マルチモーダル対話**と、文脈付きで推論を学ばせる **ScienceQA** の 2 つを Stage 2 のシナリオとして示している。

| 段階 | 学習する部品 | 凍結する部品 | データ（規模感） |
| --- | --- | --- | --- |
| Stage 1 | 線形射影 W のみ | 視覚エンコーダ・LLM | CC 由来の画像-文 約595K |
| Stage 2 | 線形射影 W ＋ LLM | 視覚エンコーダ | 合成指示 約158K |

なお視覚エンコーダの入力解像度は ViT-L/14 の **224px 系**が中心であり、336px を標準採用するのは後継の LLaVA-1.5 以降の話なので、ここでは断定を避けておく。

## GPT-4 による指示データ合成

単純なアーキテクチャを成立させたのは、データ側の工夫だ。当時、視覚言語の「指示追従」データはほとんど存在しなかった。LLaVA はこれを、**画像を一切見せずに text-only の GPT-4（または ChatGPT）へ作らせる**という発想で突破する。鍵は、画像を 2 種類の**記号表現**に翻訳して言語モデルに渡す点だ。

- **キャプション**：シーンを複数の視点から述べた文。画像の「内容」を言語で要約する。
- **バウンディングボックス**：`person:[0.681, 0.242, 0.774, 0.694]` のように、各物体の概念と空間位置を記号化する。画像の「配置」を言語で与える。

この 2 つを渡せば、GPT-4 は実画像を見なくても、あたかも画像を見ているかのような指示データを生成できる。COCO 画像をもとに、次の 3 種類が作られた。
