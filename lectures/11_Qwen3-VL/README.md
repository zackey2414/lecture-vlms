# Qwen3-VL — Technical Report（Dense/MoE・Thinking・長コンテキスト・Agent）

Qwen3-VL（arXiv:2511.21631, Qwen Team, 2025）は、Qwen2-VL / Qwen2.5-VL の系譜を継ぐ最新世代の視覚言語モデル（VLM）である。Qwen3 を言語バックボーンとし、**Dense（2B/4B/8B/32B）と MoE（30B-A3B / 235B-A22B）の二系統**、さらに推論を強化した **Thinking 版**を提供する点が最大の特徴で、ネイティブで **$256\text{K}$ トークン**級の画像・動画・テキスト混在コンテキストを扱う。本ページは、Qwen2.5-VL を既習の研究者が「前世代から何がどう拡張されたか」を腹落ちさせることを狙う。

## 全体像（まず一枚で）

Qwen3-VL の骨格は Qwen2.5-VL を踏襲した三モジュール構成である。すなわち **視覚エンコーダ（SigLIP-2、動的・ネイティブ解像度）→ MLP ベースの merger（$2 \times 2$ の視覚特徴を 1 トークンに圧縮）→ Qwen3 LLM デコーダ（Dense / MoE）**という流れだ。視覚エンコーダはデフォルトで SigLIP2-SO-400M を、小型 LLM（2B/4B）向けには SigLIP2-Large（300M）を用い、動的解像度に対応するため 2D-RoPE と絶対位置埋め込みの補間（CoMP に倣う）を組み合わせている。

土台は前世代と同じだが、報告は**三つのアーキテクチャ的アップグレード**を明示している。

- **Interleaved-MRoPE**：Qwen2.5-VL の MRoPE は埋め込み次元を時間 ($t$)・水平 ($h$)・垂直 ($w$) の塊に分割していたが、これが周波数スペクトルの偏りを生み、長尺動画理解を損なっていた。Qwen3-VL は $t$, $h$, $w$ を全次元に**均等にインターリーブ**し、各時空間軸を低周波・高周波の両帯域で一様に表現する。
- **DeepStack**：ViT の**中間層**から取り出した視覚トークンを専用 merger で射影し、**LLM の下位 3 層**に残差的に加算する。多層特徴（低レベル〜高レベル）を、コンテキスト長を増やさずに融合する。
- **テキストベースの時刻整列**：動画の時間情報を、位置符号化（T-RoPE による絶対時刻）ではなく**明示的なテキスト時刻トークン**で与える（後述）。

加えて学習目的側では、損失をサンプル単位から**平方根正規化したトークン単位**へ変更し、テキストとマルチモーダルの寄与のバランスを取っている。

## Dense と MoE、そして Thinking 版

Qwen3-VL は同一の設計思想を**二つの計算配分**で実装する。**Dense**（2B/4B/8B/32B）は全パラメータを毎トークン計算する素直な構成で、エッジや小型用途に向く。**MoE**（30B-A3B / 235B-A22B）はルータが入力ごとに**一部の専門家（expert）のみを活性化**し、総パラメータは大きいまま活性パラメータを抑える。記法 `A3B` / `A22B` は**活性パラメータ**を表し、30B-A3B は総 30B で活性 3B、235B-A22B は総 235B で活性 22B を意味する。つまり「容量は MoE で稼ぎ、推論コストは活性分に抑える」という、レイテンシ・品質トレードオフへの直接の手当てである。

各サイズには、さらに **Thinking 版**と非 Thinking 版（Instruct）が用意される。Thinking 版は長い Chain-of-Thought（CoT）を明示的にモデル化し、複雑な多段推論で大きく性能を伸ばす。学習の流れは、4 段階の事前学習（後述）を経たのち、後段で **SFT（$32\text{k} \to 256\text{k}$ 文脈）→ 強→弱の蒸留（テキストのみで LLM バックボーンを微調整）→ 強化学習（Reasoning RL に SAPO を採用 ＋ General RL）**と進み、ここで non-thinking と thinking に分岐する。Thinking 版の cold start には純テキストとマルチモーダルを概ね 1:1 で混ぜた Long-CoT データを用い、「視覚を見ないと解けない」問題だけを残すフィルタ（Qwen3-30B-nothink が画像なしで解けてしまう例を除外）など、視覚必須性を担保する工夫が入る。報告の評価でも、たとえば MMStar で 235B-A22B-Thinking が系列最高水準を示す一方、Instruct 版が RealWorldQA で Thinking 版を上回る場面もあり、**Thinking が常に最善とは限らない**点は実務上の留意点である。

## 長コンテキストと動画・時間理解

Qwen3-VL はネイティブで **$256\text{K}$ トークン**（実装上 $262{,}144$）の混在コンテキストを扱い、長文書・長尺動画の保持・検索・相互参照を狙う。この長コンテキスト能力は、事前学習を**段階的に**伸ばすことで獲得される。

| 段階 | 目的 | 学習対象 | トークン予算 | 系列長 |
|---|---|---|---|---|
| S0 | 視覚言語アライメント | merger のみ | 約 67B | $8{,}192$ |
| S1 | マルチモーダル事前学習 | 全体 | 約 1T | $8{,}192$ |
| S2 | 長コンテキスト事前学習 | 全体 | 約 1T | $32{,}768$ |
| S3 | 超長コンテキスト適応 | 全体 | 100B | $262{,}144$ |

S0 ではモダリティ間の橋渡しに集中して merger だけを学習（視覚エンコーダと LLM は凍結）し、S1 で全パラメータを解凍、S2・S3 で系列長を $8\text{K} \to 32\text{K} \to 256\text{K}$ と引き上げる。後段 SFT でも $32\text{k} \to 256\text{k}$ のカリキュラムを組み、数百ページの技術文書・教科書全体・**最長 2 時間規模の動画**を扱えるよう仕上げる。

時間理解の中核は、前述の**テキスト時刻トークン**である。Qwen2.5-VL は T-RoPE で位置 ID を絶対時刻に同期させていたが、長尺では位置 ID が疎かつ肥大化して長時間文脈の理解を損ない、また多様な fps の一様サンプリングを要して学習コストが高かった。Qwen3-VL はこれを、各フレーム群の先頭に `<3.0 seconds>` のような**整形済みテキスト文字列**を前置する方式へ置き換える（秒形式と HMS 形式の双方で学習）。これにより時間表現がより直接的・精密になり、video grounding（時間ローカライズ）や dense captioning といった時間認識タスクが扱いやすくなる。Interleaved-MRoPE による均等な周波数配置と併せ、**前世代の絶対時刻整列を継承しつつ、より頑健な長時間モデリングへ作り替えた**と位置づけられる。

## GUI Agent と 3D grounding

Qwen3-VL は「見る」だけでなく「行動する」ことへ踏み込み、知覚・推論・行動を一つの基盤に統合しようとする。データ面では、**3D／空間推論を含む正規化グラウンディング**、コード、長文書、時間グラウンディング付き動画、そして GUI エージェント相互作用が新たに整備された。

**GUI Agent**：デスクトップ・モバイル・Web を横断する画面操作データを大規模に合成・収集する。画面知覚のために要素記述・dense captioning・grounding を、エージェント能力のために**自己進化的な軌跡生成**で組んだ多段タスク軌跡を用意し、CoT による計画・意思決定・自己修正を付与する。評価は ScreenSpot Pro / OSWorld-G / AndroidWorld / OSWorld / WindowsAA などで行われ、画面操作エージェントとしての実用性が測られている。関連して、マルチモーダルな function calling や、未知エンティティを画像・テキスト検索で調べる search 行動も学習される。

**Grounding と 3D**：grounding は bounding box と point の二様式を扱い、座標系は Qwen2.5-VL から変更して **$[0, 1000]$ の正規化座標**を採用する。これは解像度やアスペクト比の違いに頑健で、後処理も簡単になる。空間理解では、絶対座標ではなく**物体・シーンに対する相対関係**（「ノートPCの左のカップ」など）、affordance（graspable/pressable/sittable 等）、行動条件付きクエリ（「モニタ後ろの本に届くにはまず何を動かすか」）を学習し、「どこに」だけでなく「どうやって／何ができるか」に答えられるようにする。**3D grounding** は、単一の単眼画像から **$9\text{-DoF}$ の 3D バウンディングボックス**（位置・姿勢・寸法）を推定する専用データで学習し、Omni3D に倣って仮想カメラ座標系へ統一する。評価は ARKitScenes / Hypersim / SUNRGBD などで行われる。あわせて OCR は対応言語を 10 から **39 言語**へ拡張し、自前テストで **39 言語中 32 言語**が 70% 超の精度に達するなど、多言語・文書理解も底上げされている。

## まとめと、読解後に答えたい問い

Qwen3-VL は、Qwen2.5-VL の三モジュール構成（動的解像度・SigLIP-2・絶対時刻 mRoPE）を土台に、**Interleaved-MRoPE・DeepStack・テキスト時刻トークン**の三点で時空間モデリングと視覚整列を作り替え、**Dense/MoE の二系統＋Thinking 版**、**$256\text{K}$ 級の長コンテキスト**、**GUI Agent**、**3D grounding**、**多言語 OCR** へと知覚と行動を広げた世代である。

読解後に自分の言葉で答えたい問い:

- Interleaved-MRoPE は、Qwen2.5-VL の MRoPE の何（周波数スペクトルの偏り）を、どう（t/h/w の均等インターリーブで）解消したか。なぜ長尺動画で効くのか。
- DeepStack が「コンテキスト長を増やさずに」多層視覚特徴を融合できるのはなぜか。LLM のどの層へ注入されるか。
- 動画の時間を「絶対時刻の位置符号化」から「テキスト時刻トークン」へ変えた利点と代償（文脈長の増加）は何か。
- MoE の `A3B` / `A22B` 表記が指すものは何か。Dense と比べた速度・品質トレードオフをどう説明するか。
- Thinking 版が常に最善でないのはなぜか（RealWorldQA で Instruct が上回る例）。タスク特性とどう結びつくか。
